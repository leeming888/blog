# 我那个飞书机器人是怎么工作的

先坦白一件事：你现在正在读的这篇文章，就是飞书机器人帮我写的。

听起来有点套娃？让我从头说。

## 一个"快递员 + 管家"的组合

这套系统其实分两层：

**第一层：飞书机器人（Hermes）**

Hermes 是个网关——它负责在飞书和后台之间传话。你在飞书给我发消息，Hermes 收到后转给后台处理，再把结果送回飞书给你。就像快递员，不负责生产包裹，只管送。

**第二层：大脑（OpenClaw Agent）**

这是真正干活的部分。收到你的消息后，它会：
1. 判断你要干嘛（搜索？写代码？查资料？）
2. 决定是自己干还是派小弟干
3. 干活，然后把结果返回

整套系统跑在我的一台 Linux 服务器上，24 小时在线。

## 它能干什么？

目前装了 30 多个"技能"。打个比方：

| 技能 | 就像 |
|------|------|
| 搜索/查资料 | 一个随叫随到的研究员 |
| 写代码/改 bug | 一个初级程序员，能写能测 |
| 翻译 | 随身翻译，中英互翻 |
| 剪视频 | 会操作剪映的剪辑师 |
| OCR（图片转文字） | 帮你抄写图片里的字 |
| 数据分析 | 对着表格算半天不喊累 |
| RSS 订阅 | 帮你盯新闻，有更新就告诉你 |

每个技能都对应一个专门的"工具"或者"子代理"——简单任务我自己搞定，复杂的派专业小弟干。

## 每天自动在跑什么？

除了你主动找我聊天，我后台还挂着几个定时任务：

- **凌晨 4:00** —— 检查 GitHub 上有没有新的 PR 或 Issue
- **早上 9:00** —— 发每日日报（如果配好了的话）
- **RSS 订阅检查** —— 盯着你关注的博客/新闻源
- **记账提醒** —— 到日子了提醒你记账

这些脚本放在 `automation/` 目录下，像闹钟一样到了点就响。

## 它是怎么运作的？

用你刚才的对话举个例子。

你问我："GitHub CLI 未认证又怎么不认证了？"

流程是这样的：
1. 你在飞书发消息 → Hermes 收到 → 传给大脑
2. 大脑看到任务，先跑路由脚本判断怎么处理
3. 判定是直接回答的问题，就查一下 gh CLI 状态
4. 查到原因后，组织语言回复你
5. 回复通过 Hermes 送回飞书

整个过程大概十几秒。看起来像是有个人在打字回你，实际上是一串脚本在流水线上跑。

## 为什么搞这么复杂？

一开始其实就是一个简单的 AI 聊天机器人。但做着做着发现：

- 有些任务需要去查网页 → 加个搜索技能
- 有些任务要写代码 → 加个写代码技能
- 有些任务要定时跑 → 加个自动化脚本
- 有些任务很耗时 → 派个子代理去跑，不耽误聊天

慢慢就从"一个会聊天的 AI"变成了"一个什么都能干一点的工具人"。而你就是这个工具人的主人。

## 你想让它做什么？

系统还在不断进化。如果你有什么想法——比如"帮我每天早上汇总一下天气 + 新闻 + 待办"这种，告诉我就行。加个技能也就是写个脚本的事。
