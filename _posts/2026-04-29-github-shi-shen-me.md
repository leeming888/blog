# GitHub 是什么——给完全没接触过的人

如果你刚学编程没多久，或者只是听说过"GitHub"这个词但完全不知道它是干嘛的，这篇文章就是写给你的。

## 一个朴素的困境

想象一下这个场景：你在写一篇很重要的论文，在电脑上改了又改。一开始叫"论文_v1.docx"，过了一周变成了"论文_v2.docx"，然后是"论文_最终版.docx"、"论文_真的最终版.docx"、"论文_打死也不改了版.docx"……

是不是很熟悉？

程序员写代码的时候面临同样的问题，而且更复杂——因为代码不是一个人写的，可能是十个人、一百个人同时在改同一批文件。你怎么知道谁改了哪里？改错了怎么回溯？大家的版本怎么合并到一起？

这时候就需要一个"管家"。

## Git：版本管家

Git 就是这个管家的名字。它的工作很简单：

- **记录每一次修改**——谁、什么时候、改了哪些地方，清清楚楚
- **随时可以回头**——改坏了？没关系，退回上一个正常的版本就行
- **支持多人协作**——你改你的，我改我的，最后合并起来

你可以把 Git 理解成**一个超级好用的"修改历史"面板**，比 Word 的"修订"模式强大一百倍。

## GitHub：社交化的代码仓库

那 GitHub 又是什么呢？

Git 是你电脑上的管家。但如果你的代码只想自己看，那管家就够了。如果你想**把代码放到网上**，让其他人也能看到、使用、甚至帮你改进，你就需要一个"云端的家"——这就是 GitHub。

打个比方：

Git 像你书桌上的文件夹和笔记本，帮你整理好自己的东西。GitHub 像**一个巨大的图书馆**——你可以把自己的笔记本放上去（公开的），别人可以翻阅、借阅。如果发现你写错了一个字，人家还能直接在本子上改，然后告诉你"我帮你订正了一下"。

更妙的是，图书馆里的每一本笔记本都有**完整的历史记录**——每一页每一行是谁写的、什么时候写的、后来改成了什么，全都查得到。

## 不只是存代码的地方

GitHub 上现在有超过一亿个项目，从一个人写的小工具到 Linux 内核、Python 语言、Vue.js 这类全世界都在用的顶级项目，都在上面。

说几件 GitHub 能干的事：

- **发现好项目**——想看别人怎么用 Python 写爬虫？搜一下就有成千上万的开源项目可以学习
- **提交建议**——你发现某个开源软件有个 bug，可以直接提交修改。如果被采纳了，你的名字就会出现在这个项目的贡献者名单里
- **搭建网站**——就像我这个博客一样，GitHub Pages 可以免费托管网站
- **项目管理**——可以用 Issue 来跟踪任务和 Bug，用 Wiki 写文档，用 Discussions 讨论问题

## 简单来说

- **Git** = 你电脑上的时光机，记录代码的每一次变化
- **GitHub** = 这个时光机连接到了云端，让全世界的人可以一起协作

如果你刚开始学编程，注册一个 GitHub 账号、把代码推上去、看看别人的项目，就是迈向"正规军"的第一步。不用担心不会用——所有人都是从"git push 失败了怎么搞"开始的。

加油，欢迎来到这个精彩的世界。
