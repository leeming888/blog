# Git 入门——别怕命令行

如果你看到终端里黑底白字就头大，这篇文章就是写给你的。

## 先搞明白 Git 到底在干嘛

把 Git 想象成一个游戏存档系统。你打游戏的时候会存档对吧？存了档，打不过 boss 可以读档重来，打出了隐藏结局可以回去走另一条线。

Git 就是代码的存档系统。

每次你"存档"（Git 里叫 **commit**），它都会记下来：
- 这次改了什么文件
- 谁改的
- 什么时候改的
- 改的什么东西

然后你就可以随时"读档"——回到任何一个历史版本。

## 你只需要记住 5 个命令

说实话，日常 90% 的场景就这几个命令：

### 1. `git clone` —— 把网上的代码下载到本地

```
git clone https://github.com/leeming888/blog.git
```

这行命令把网上的博客仓库整个复制到你电脑上。一次操作，全套文件 + 全部历史版本都下来。

就像从图书馆借了一本书——书的内容、页码、甚至以前的修改痕迹全都有。

### 2. `git add` —— 告诉 Git "我要存这些文件"

你改了几行代码，跑了一下没问题。现在你想存档。

```
git add 文件名
# 或者一次加所有改过的文件
git add .
```

`add` 的意思是"把文件放进暂存区"——就像把要存档的进度先选中，还没真正存。

### 3. `git commit` —— 真正存档

```
git commit -m "修复了登录按钮点击没反应的问题"
```

`commit` 就是正式存档了。`-m` 后面跟的是**存档说明**——告诉未来的自己或队友，这次改了什么。

**写 commit 信息的小窍门：** 用"做了什么事"的句式，比如"修复了 XX 问题"、"新增了 XX 功能"、"删除了没用的测试代码"。不用写长篇小说，几秒钟能看懂就行。

### 4. `git push` —— 把存档上传到云端

```
git push
```

你在本地存了档，别人看不到。`push` 把存档同步到 GitHub 上，所有人都能看到了。

就像你单机打通了关，然后上传云存档，顺便晒一晒成就。

### 5. `git pull` —— 把别人的更新同步下来

```
git pull
```

队友改了几行代码推上去了，你想拿到最新版本？`pull` 一下就行。

## 一次完整的日常操作

写了代码 → `git add .` → `git commit -m "加了什么"` → `git push`

就三步，每天重复。习惯了比点鼠标还快。

## 为什么要用命令行？

代码编辑器（VS Code 之类的）确实有图形界面可以点按钮提交代码。但命令行有几个好处：

- **快** —— 手不用离开键盘，一行命令搞定
- **统一** —— 不管你在哪台电脑上工作，操作一模一样
- **看得见过程** —— 报错了你能看到错误信息，方便查

**给新手的建议：** 先用命令行练几天，肌肉记忆了再用图形界面——这样就算图形界面崩了，你也不慌。

## 最后说一句

Git 命令行就像骑自行车。第一次上车觉得要摔，摔几次就习惯了，然后发现它比走路快太多了。

不用一次背完所有命令。记住 `add → commit → push` 这三板斧，你就可以上路了。遇到问题？搜一下就行——每个程序员都是从"git push 失败了"的恐慌中成长起来的。
